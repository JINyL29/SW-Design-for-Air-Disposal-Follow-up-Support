# 드론 전기 계통 고장 진단 시스템
## 시스템 개요 보고서

---

## 1. 시스템 목적

본 시스템은 드론의 전기 계통(배터리, 전력 분배, 비행 제어, 통신 등)에서 발생하는 **전압 이상을 실시간으로 진단**하고, 고장 원인을 분석하여 **정비 절차를 제공**하는 소프트웨어입니다.

전기 계통 고장은 드론의 비행 안전과 직결되며, 초기 단계에서 감지하지 못하면 연쇄 고장으로 이어질 수 있습니다. 본 시스템은 이러한 연쇄 고장의 인과관계를 파악하고, 운용자가 신속히 대응할 수 있도록 지원합니다.

## 3. 드론 전기 계통 구성 (배선도 토폴로지)

### 3.1 전력 공급 흐름

```
BAT ──→ PDB ──→ ESC ──→ MOT
              │
              └──→ BEC ──→ FC  ──→ GPS
                        │
                        ├──→ TEL
                        │
                        └──→ CAM
```

- **실선**: 전력 공급 경로
- **점선**: PWM/UART 신호 경로 (ESC↔FC, FC↔TEL, FC↔ESC)

### 3.2 배선 라인 종류

| 라인 종류 | 색상 | 연결 경로 |
|---|---|---|
| 44V 메인 파워 | 빨강 | BAT→PDB, PDB→ESC, PDB→BEC |
| 12V BEC 출력 | 보라 | BEC→FC, BEC→TEL, BEC→CAM |
| 5V 조정 전압 | 갈색 | FC→GPS |
| UART 신호 | 초록 | FC↔TEL |
| PWM 신호 | 회색 | ESC↔MOT, FC↔ESC |

### 3.3 연쇄 영향 관계

| 고장 부품 | 직접 영향 하위 부품 |
|---|---|
| BAT | ESC, BEC (그리고 각각의 하위 전체) |
| ESC | MOT |
| BEC | FC, TEL, CAM |
| FC | GPS |
| TEL, CAM, GPS, MOT | 없음 (leaf node) |

## 5. 전압 상태 분류 기준

### 5.1 전압 구간 (Zone)

| 구간 | 조건 | 설명 |
|---|---|---|
| normal | critical_low ≤ V ≤ normal_max | 정상 |
| warning | critical_low ≤ V < normal_min | 경고 (저전압 주의) |
| critical | V < critical_low | 임계 (즉시 조치 필요) |
| OV | V > normal_max | 과전압 |

### 5.2 전압 상태 (VoltageState)

| 상태 | 설명 |
|---|---|
| NORMAL | 정상 전압 범위 |
| UNDER_VOLTAGE | 저전압 (경고 또는 임계 구간) |
| OVER_VOLTAGE | 과전압 |
| RAPID_DROP | 급격한 전압 강하 |
| NO_DATA | 전압 데이터 없음 |
| SENSOR_ERROR | 센서 오류 |

### 5.3 심각도 결정 방식

심각도는 **전압 zone 심각도**와 **부품별 고장코드 심각도** 중 **더 높은 값**을 채택합니다.

```python
최종 심각도 = max(zone 심각도, 고장코드 심각도)
```

| 전압 zone | zone 심각도 |
|---|---|
| critical UV/OV | CRITICAL |
| warning UV/OV | HIGH |
| NO_DATA / SENSOR_ERROR | HIGH |
| RAPID_DROP | CRITICAL |
| normal | LOW |

**BAT, ESC, BEC, MOT(UV)**는 고장코드 DB에 CRITICAL로 설정되어 있어 경고 구간이더라도 항상 CRITICAL로 판정됩니다. 이는 이들 부품의 고장이 하위 전체에 연쇄 영향을 주기 때문입니다.

### 8.2 2단계 캐스케이딩 (6~10번)

| # | 시나리오명 | 고장 부품 |
|---|---|---|

| 12 | BEC 과전압 → FC·TEL·CAM·GPS 과전압 손상 | BEC → FC, TEL, CAM, GPS |
| 13 | TEL·CAM 동시 고장 (BEC 출력 라인 부분 단선) | TEL, CAM |

### 8.4 전체 캐스케이딩 (14~16번)

| # | 시나리오명 | 고장 부품 |
|---|---|---|
| 14 | BAT 완전 방전 — 전 계통 저전압 | BAT → 전부품 |
| 15 | BAT 과충전 — 전 계통 과전압 | BAT → 전부품 |
| 16 | BAT 급격한 전압 강하 — 비상 착륙 | BAT → 전부품 |

### 8.5 복합 시나리오 (17~19번)

| # | 시나리오명 | 고장 부품 |
|---|---|---|
| 17 | PDB 접촉 불량 — ESC·BEC 동시 저전압 | ESC, MOT, BEC, FC, TEL, CAM, GPS |
| 18 | ESC 급격한 전압 강하 → MOT 추력 불안정 | ESC → MOT |
| 19 | 전 계통 정상 | 없음 (전부품 정상) |

### 8.6 시나리오 전압 생성 방식

- **저전압 (UV)**: 70% 확률로 임계 구간, 30% 확률로 경고 구간에서 랜덤 값 생성
- **과전압 (OV)**: 정상 범위 초과 구간에서 랜덤 값 생성
- **급강하 (RAPID_DROP)**: 임계값 직전 구간 랜덤 값 생성 (BAT: 35.5~37.5V, ESC: 37.5~39.5V)
- **캐스케이딩 시각화**: 500ms 간격으로 순차적으로 고장 전파 → 배선도에서 실시간 확인 가능

## 9. 수동 진단 및 인과관계 전파

### 9.1 수동 진단 방법

시뮬레이션 UI에서 다음 두 가지 방법으로 수동 진단이 가능합니다.

1. **전압 직접 입력**: 원하는 전압값을 입력 후 "진단 실행" 클릭
2. **시나리오 버튼**: Normal / Under-Voltage / Over-Voltage / No Data / Sensor Error / Rapid Drop 중 선택

### 9.2 부모 부품 정상화 시 하위 전파

부품이 NORMAL로 진단되면 하위 부품들이 자동으로 재진단됩니다.

```
사용자: BEC → 정상 전압 입력 → 진단 실행
  ↓ BEC = NORMAL
  ↓ _propagate_normal("BEC") 호출 (300ms 간격)
  ↓ FC 재진단 → cascade 고장이었다면 NORMAL 복귀
  ↓ TEL 재진단 → cascade 고장이었다면 NORMAL 복귀
  ↓ CAM 재진단 → cascade 고장이었다면 NORMAL 복귀
  ↓ (FC NORMAL → GPS 재진단 → NORMAL 복귀)
```

**cascade vs 독립 고장 구분:**
- **cascade 고장** (시나리오로 설정됨): 부모 정상화 시 → NORMAL 복귀
- **독립 고장** (사용자가 수동 주입): 부모 정상화 시 → FAULT 유지

### 9.3 부모 부품 고장 시 하위 전파

부품이 FAULT (UV/OV)로 수동 변경되면 하위 부품도 연쇄 영향을 받습니다.

```
사용자: BEC → Over-Voltage 주입
  ↓ BEC = FAULT (OV)
  ↓ _propagate_fault("BEC", OVER_VOLTAGE) 호출
  ↓ FC → OV 전압으로 재진단 → FAULT
  ↓ TEL → OV 전압으로 재진단 → FAULT
  ↓ CAM → OV 전압으로 재진단 → FAULT
  ↓ (FC FAULT → GPS → FAULT)
```

**중요**: `_propagate_fault` 호출 시 `_last_diag`(건강 전압)는 변경하지 않으므로, 이후 BEC가 정상화되면 하위 부품들이 원래 건강 상태로 복구됩니다.

## 10. 대시보드 UI 기능

### 10.1 좌측 패널

- **전체 부품 / 정상 / 경고 KPI 카드**: 실시간 부품 상태 집계
- **시스템 건강도 게이지**: 정상 부품 비율 (%) 표시, 80% 이상 녹색 / 50% 이상 노랑 / 이하 빨강
- **심각도 분포 차트**: CRITICAL/HIGH/MEDIUM/LOW 별 부품 수 막대 표시
- **진단 파이프라인**: 진단 실행 시 6단계 파이프라인 애니메이션

### 10.2 중앙 패널

- **전기 배선도 토폴로지**: 부품 상태에 따라 노드 색상 실시간 업데이트
  - 정상: 녹색 / 저전압: 주황 / 과전압: 빨강 / 경고: 노랑 / 데이터없음: 분홍
  - 고장 부품에 `!` 배지 표시
  - PDB는 BAT 상태 자동 반영
- **부품 상태 테이블**: 부품명, 모델, 정격 전압, 현재 전압, 상태, 고장코드, 심각도, 정비 조치 표시
- **시스템 로그**: 실시간 진단 이력 (색상 코딩: CRITICAL=빨강, HIGH=주황, MEDIUM=파랑, NORMAL=초록)

### 10.3 우측 패널 (활성 고장 상세 정보)

- **심각도 필터 버튼**: 전체 / 긴급 / 높음 / 중간 필터
- **고장 카드**: 부품명, 고장 코드, 측정 전압, 상태, 심각도, 감지 시각, 정비 절차
- **전선 점검 정보 배지**: AWG 규격(파랑), 커넥터 종류(XT90=빨강/XT30=주황/GH1.25=파랑), 정격 전류

### 10.4 알람 배너

- 고장 발생 시 타이틀 바 아래에 심각도별 컬러 배너 자동 표시
- 가장 심각한 고장 코드 / 부품명 / 상태 / 추가 고장 수 표시
- 초기화 시 자동 숨김

### 10.5 전선 점검 경고 스트립

- **동일 부품 3회 연속 고장** 시 황색 경고 스트립 자동 표시
- AWG 규격 / 커넥터 / 정격 전류 정보와 함께 전선 손상 점검 권고
- 초기화 시 자동 숨김

- 고장 발생 부품: `⚠ CRITICAL` 등 심각도 배지 표시
- 전압 직접 입력 필드 및 정상 범위 게이지 표시

### 11.2 고장 시나리오 주입 패널

- 6가지 시나리오 버튼: Normal / Under-Voltage / Over-Voltage / No Data / Sensor Error / Rapid Drop
- 클릭 시 해당 전압 자동 입력 및 진단 즉시 실행

### 11.3 진단 결과 프리뷰

- 진단 상태 / 고장 코드 / 심각도 / 편차(%) / **전선 규격** 5개 카드 표시

### 11.4 우측 컨트롤 패널

- **진단 실행** 버튼: 선택 부품 단독 진단 + 하위 부품 자동 전파
- **전체 부품 진단** 버튼: 19개 시나리오 중 랜덤 선택, 캐스케이딩 시각화
- **초기화** 버튼: 전체 상태 리셋
- **정격 전압 참조** 패널: 선택 부품의 정상/경고/임계 전압 범위 표시
- **시뮬레이션 로그**: 타임스탬프 포함 진단 이력

## 12. 주요 기술 구현 사항

### 12.1 연속 고장 감지 (전선 점검 경고)

```
전체 부품 진단 실행마다 부품별 연속 고장 횟수 추적
동일 부품이 3회 연속 고장 → wire_warning 시그널 발송
→ 대시보드에 전선 점검 경고 황색 스트립 표시
초기화 시 연속 횟수 리셋
```

### 12.2 마지막 진단 시각

- 타이틀 바 우측에 `마지막 진단: HH:MM:SS — 부품ID` 실시간 표시

### 12.3 Ctrl+C 종료 처리

- Qt 이벤트 루프의 SIGINT 무시 문제를 200ms QTimer로 해결
- 터미널에서 Ctrl+C 한 번으로 즉시 종료 가능
